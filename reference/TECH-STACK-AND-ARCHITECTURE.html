3Design principles (the non-negotiables)

  1. The deterministic core stays pure. No I/O, no LLM, no framework inside the verdict path. It's a pure function: (study, answers) → verdict + trace. Everything else wraps it.
  2. The LLM is confined to extraction. An LLM mistake changes one variable and is visible in the trace; it never decides eligibility.
  3. One core, many channels & surfaces. CLI, web, SMS, voice all call the same engine through one interface.
  4. Schema is the single source of truth. One typed, runtime-validated schema for study.json, questions, verdicts, and conversation records.
  5. Restructure, don't rewrite. Port the proven logic verbatim into modules; wrap with types, tests, and infra around it.
  6. Compliance is a gate, not a phase. No PHI on a live channel until HIPAA/TCPA review passes.

8LLM extractor — Anthropic Haiku 4.5

The extractor turns a patient's free-text reply into a structured value. It is the only LLM in the request path, and it never decides eligibility.

Haiku 4.5
model id claude-haiku-4-5 — right tier for cheap, high-volume per-turn extraction
$1 / $5
per 1M input / output tokens · 200K context
~0.1×
cached-prefix read cost — system prompt + question schema cached across turns
strict
structured tool-use / output_config.format → guaranteed {value, confidence, needs_clarification}
  • Model. Haiku 4.5 (claude-haiku-4-5) — extraction is a small, well-scoped task; reserve larger models for nothing here. ($1/MTok in, $5/MTok out; 200K context.)
  • Prompt caching. The stable prefix (extractor system prompt + the question's variable/type/choices) is identical across turns and patients — cache it (cache_control: ephemeral) and pay ~0.1× on reads. Only the patient's reply is volatile, placed after the breakpoint. Verify with usage.cache_read_input_tokens; keep the prefix byte-stable (no timestamps).
  • Structured output. Use a tool with strict: true (or output_config.format with a JSON schema) so the response is always { value, confidence, needs_clarification } — no parsing, no free-form drift.
  • Determinism preserved. The extractor feeds a value; the verdict is still pure code. Low confidence → the engine re-asks a clarifying question; the branch logic never sees ambiguity.
  • Resilience. Timeout + fall back to the rule extractor; an extractor error is localized to one variable and visible in the trace.
  • Compliance. HIPAA-eligible Anthropic API under a signed BAA; minimum-necessary data to the extractor; redact identifiers where possible.
Cost intuition. A screening turn sends a tiny volatile reply against a cached prefix, so per-turn cost is dominated by a few hundred cached-read tokens at ~0.1× plus a short structured output — fractions of a cent per turn. Caching + Haiku is what makes per-turn LLM extraction economical at scale.

9Scalability path

  • Stateless API + Redis sessions → scale the API horizontally; no sticky sessions.
  • Pure engine → trivially parallel; batch scoring is embarrassingly parallel.
  • Extraction is the cost/latency lever → Haiku + prompt caching; rule-first fast path, LLM only when the rule extractor is unsure; cache hit-rate is a monitored metric.
  • Postgres → read replicas for funnel/reporting; partition conversations by study/date as volume grows.
  • Channels behind one adapter → add SMS/voice without touching the core; per-study routing handled at the API edge.
  • Multi-study, multi-tenant → study config is data, not code; a new study is a row, not a deploy (mirrors Alleviate's authoring model, but with our eval gate in front).
